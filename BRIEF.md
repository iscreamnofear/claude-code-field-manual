We're building a downloadable e-book called the **Claude Code Field Manual**. It's a practical guide for **non-technical professionals** (marketing, sales, customer success, ops) who want to use Claude Code and Cowork to build real things without waiting for a developer.

## Target audience

Non-technical professionals who:
- Have never opened a terminal
- Don't know what "git" or "npm" means
- Work in marketing, sales, customer success, ops, or leadership
- Have ideas and projects stuck in dev backlogs
- Want to build things themselves: dashboards, websites, automations, internal tools

## Design direction

This should feel like a premium tech e-book. Not corporate. Not developer-bro. Not a whitepaper. Think: Intercom's guides, Stripe's documentation aesthetic, or a well-designed indie tech book. Clean, modern, with enough personality to be memorable.

- Illustrations should be consistent in style across the whole book
- Chapter openers should feel like section breaks, not just bigger text
- Code blocks and callout boxes need visual polish (the content/colors are defined, the rendering needs design treatment)
- The cover should be distinctive and shareable (people screenshot covers)
- Page layout should breathe ‚Äî generous margins, whitespace, pull quotes
