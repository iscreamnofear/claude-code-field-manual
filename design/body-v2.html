The five rules for prompting

You don't need to learn "prompt engineering." You need to communicate clearly. If you can write a decent email to a contractor, you can prompt Claude.

1. Start with what you want, not how to build it

You're the client, not the architect. Describe the outcome. "I need a landing page for our webinar" beats "Create an HTML file with a hero section using flexbox."

2. Include the context Claude can't guess

Claude doesn't know your company, your brand colors, your audience, or your deadline. Include anything that would affect the result.

Worth Knowing

Claude remembers everything in the current session but starts fresh each time you open a new one. If you have brand guidelines, product descriptions, or style preferences — paste them in at the start of every session.

3. Be specific about the output

Vague: "Make it look good." Specific: "Use our brand colors. No more than three sections. Include a call-to-action button above the fold."

4. Say what you DON'T want

Claude is enthusiastic. Left unsupervised, it will add features you didn't ask for. If you want something simple, say so.

5. Ask for a plan before execution

Always start with: "Before building anything, explain your plan." Review the plan. Adjust. Then give the green light.

Save Yourself 45 Minutes

Keep a document with prompts that worked well. When you find a phrasing that gets great results, save it. You'll reuse it.

prompt-template.txt
"I need [what you're building]. The audience is [who will use it]. The goal is [what it should accomplish]. It needs to include: [specific features]. Style: [describe the look and feel]. Brand colors: [hex codes if you have them]. Do NOT: [things you want to avoid]. Before building, explain your plan step by step."

How to give feedback mid-project

Instead of thisSay this
"This isn't right""The headline is too small and the button should be blue, not gray"
"Make it better""The spacing between sections feels cramped. Add more white space"
"I don't like it""The overall style is too playful. Make it more corporate and minimal"
"Fix it""The registration link goes to the wrong URL"

Be specific. Point to the exact thing. Describe the exact change.

Reality Check

"Try again" is the most useless thing you can tell Claude. It will do the same wrong thing with slightly different words. Interrupt, redirect, reframe.

This Will Break Things

Never switch to Act mode unless you understand what your project contains. Act mode is for experienced users who have set up proper safeguards.

I Learned This The Hard Way

The first time I used Act mode, Claude helpfully reorganized my entire project folder structure "for clarity." It took an hour to undo.

Do Not Skip This

Rewind only undoes file changes on your computer. If Claude did something "out in the world," rewind cannot take that back.

Copy This Exactly

"Create a .claudeignore file in my project folder that blocks access to any file starting with .env, any folder called secrets/, and any file ending in .key"

Jargon Buster

Diff view — A display that shows what changed in a file. Green lines were added. Red lines were removed.

Project 01

Turn a Spreadsheet into a Real Dashboard

Your boss wants a dashboard. You have the data in Excel. Let's skip the six-week dev request.

DetailInfo
WhoMarketing ops, sales ops, CS leads, anyone reporting metrics
What you getA live dashboard with charts and filters that opens in any browser
ToolClaude Code Desktop
Time30–60 minutes including styling back-and-forth

First prompt (copy this):

Copy This Exactly

I have a spreadsheet with our monthly sales data. I want to turn it into a dashboard with: a bar chart showing revenue by month, a line chart showing deal count trend, a dropdown to filter by region, and a summary row at the top showing total revenue, average deal size, and number of deals. I have zero coding experience. Walk me through the plan before building anything.

What happens next

Claude will ask you to share the spreadsheet (drag it in or attach it). It will look at the columns and data types, then propose a plan. Review the plan. Say "Go."

Worth Knowing

Column names matter. If your spreadsheet has a column called "Rev" and you said "revenue" in your prompt, Claude might get confused. Use the exact column names from your file.
